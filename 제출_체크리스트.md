### 절대 포함하면 안 되는 파일/폴더

| 제외 대상 | 이유 |
|---|---|
| `build/` 전체 | CMake 빌드 산출물 |
| `bin/` 전체 | 컴파일된 바이너리 (exe, .o, .a 등) |
| `*.o`, `*.obj` | 오브젝트 파일 |
| `*.exe`, `*.dylib`, `*.so` | 실행 파일 / 라이브러리 |
| `CMakeCache.txt`, `CMakeFiles/`, `cmake_install.cmake` | CMake 캐시 |
| `.vscode/`, `.idea/`, `*.xcodeproj` | IDE 설정 |
| `.DS_Store`, `Thumbs.db` | OS 메타데이터 |
| `*.log` | 로그 파일 |
| `.gitignore` | git 메타 |

> **.env.server 보안 주의**: 현재 `.env.server`에 DB 비밀번호(`DeadZone2026!@`)가 평문으로 포함되어 있습니다.
> 제출용 ZIP에는 **비밀번호를 지우거나 placeholder로 교체**한 뒤 포함하거나, 아예 제외하세요.

### ZIP 만들기 명령 (macOS 터미널)

```bash

cd ~/Desktop

zip -r DeadZone_submit.zip gameprogramming \
    --exclude "gameprogramming/build/*" \
    --exclude "gameprogramming/bin/*" \
    --exclude "gameprogramming/.git/*" \
    --exclude "gameprogramming/**/*.o" \
    --exclude "gameprogramming/**/*.a" \
    --exclude "gameprogramming/**/*.dylib" \
    --exclude "gameprogramming/**/*.exe" \
    --exclude "gameprogramming/**/CMakeCache.txt" \
    --exclude "gameprogramming/**/CMakeFiles/*" \
    --exclude "gameprogramming/**/.DS_Store" \
    --exclude "gameprogramming/.gitignore"
```

### 빌드 방법 README (ZIP 안에 포함 권장)

```
의존성 (macOS Homebrew):
  brew install sdl2 sdl2_image sdl2_mixer sdl2_ttf enet cjson mysql

  cmake --build build -j$(nproc)

  ./build/bin/DeadZoneServer

  ./build/bin/DeadZoneClient
```

### 보고서 목차 (권장 구성)

#### 1장. 게임 소개 (0.5~1p)
- 게임 이름: **DeadZone**
- 장르: 탑다운 좀비 서바이벌 멀티플레이어
- 핵심 컨셉: 4팀 × 2인, 좀비로부터 살아남아 탈출존에서 탈출
- 팀 구성 및 역할 분담

#### 2장. 게임 기획 개요 (1~1.5p)
- 맵 구성: City Ruins 200×200타일, 3구역(주거/쇼핑몰/산업)
- 핵심 메카닉:
  - 노이즈 시스템 (행동마다 좀비 어그로 반경 상이)
  - 팀 연합/배신 시스템
  - 건설 시스템 (바리케이드, 포탑, 가시 트랩)
  - 화염 전파 시스템 (BFS)
  - 5분 후 탈출존 오픈 + 5초 채널링 탈출
- 아이템/무기 등급 체계 (normal → enhanced → rare → unique)

**그림 추천**: 맵 구역 다이어그램, 아이템 등급 표

#### 3장. 기술 아키텍처 (1.5~2p)
- 전체 구조도: Client ↔ ENet UDP ↔ Server ↔ MySQL
- ECS(Entity Component System) 자체 구현
  - `World`, `ComponentPool<T>`, Dirty Flag 동기화
  - 컴포넌트 9종 목록 및 역할
- 네트워크 권한 모델
  - 서버 권한(Server Authority)
  - 클라이언트 사이드 프리딕션 + 서버 롤백
  - CHAN_RELIABLE / CHAN_UNRELIABLE 채널 분리
- 서버 고정 스텝(Fixed-step tick) 구조

**그림 추천**: ECS 구조도, Client-Server 통신 흐름도

#### 4장. 핵심 시스템 구현 (2~3p)
각 시스템별 간략 설명 + 코드 스니펫 또는 다이어그램

| 시스템 | 핵심 내용 |
|---|---|
| ZombieAI FSM | Idle→Alert→Chase→Frenzy 상태전이, Brute/Runner/Shambler 3종 |
| NoiseSystem | 액션별 noiseRadius, 좀비 FSM 트리거 |
| FireSystem | BFS 화염 전파, 포탑 연쇄 폭발 |
| CombatSystem | AABB 근접 판정, 출혈 DoT, 배신 감지 |
| AllianceSystem | 핸드셰이크 연합, 배신 즉시 해제 |
| ExtractionSystem | 300초 대기 후 존 오픈, 5초 채널링 |
| BuildSystem | 바리케이드·포탑 배치 및 자동 공격 |
| Database | MySQL 계정/인벤토리/통계 영속화 |

**그림 추천**: ZombieAI FSM 다이어그램, FireSystem BFS 전파 그림

#### 5장. 클라이언트 구현 (1p)
- SDL2 렌더링 파이프라인
- Y-sort 기반 깊이 렌더링 (건물 앞/뒤)
- 120도 FOV (Fog of War) 구현
- 파티클 시스템 (혈흔, 총구화염, 탄피, 힐 이펙트)
- 드래그 앤 드롭 인벤토리 UI
- 카메라 쉐이크 + 히트 플래시 연출

**그림 추천**: 게임 플레이 스크린샷, HUD/인벤토리 UI 스크린샷

#### 6장. 결과 및 시연 (0.5~1p)
- 실행 환경 및 빌드 방법 요약
- 구현 완료 기능 목록
- 미완성/한계 사항
- 소감 및 배운 점
