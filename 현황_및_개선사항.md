### ⚠️ 구현됐지만 버그/불완전한 기능

아래 항목들은 코드는 존재하지만 실제 동작에 문제가 있는 것들입니다.

#### BUG-1. `onDamage` 에서 배신 감지 중복 실행
- **파일**: `src/server/GameServer.cpp` L50~64, L525~537
- **문제**: `m_combat.onDamage` 콜백 등록 시(L50)와 `onDamage()` 메서드 내부(L525) 에서 `handleBetrayal()`을 **두 번** 호출합니다. 연합이 두 번 끊기거나 브로드캐스트가 중복으로 나갑니다.
- **수정**: `GameServer::onDamage()` 내부의 betrayal 처리(L525~537) 블록을 삭제하면 됩니다. 람다 콜백에 이미 있습니다.

#### BUG-2. 친팀 피해(Friendly-fire) 가드 주석 처리됨
- **파일**: `src/server/systems/CombatSystem.cpp` L213~215
- **문제**: friendly-fire 방지 코드가 `//`로 주석 처리되어 있어 **같은 팀끼리 서로 피해를 줍니다**. 헤더 주석은 "Same team → damage is blocked"라고 적혀 있지만 실제로는 허용 중입니다.
- **수정**: 기획 의도 확인 후 주석을 해제하거나, "팀 PvP 허용" 설계라면 헤더 주석을 업데이트하세요.

#### BUG-3. `triggerExplosion`에서 데미지 이벤트 이중 발송
- **파일**: `src/server/systems/CombatSystem.cpp` L259~261
- **문제**: `applyDamage()` 내부에서 이미 `m_onDamage(res)`를 호출하는데, `triggerExplosion()`이 그 반환값으로 `m_onDamage(result)`를 **한 번 더** 호출합니다. 폭발 데미지 이벤트가 클라이언트에 2번 전송됩니다.
- **수정**: `triggerExplosion()` L261의 `if (m_onDamage) m_onDamage(result);` 줄을 삭제하세요.

#### BUG-4. `onJoinMatch` 스타터 아이템 itemID 불일치
- **파일**: `src/server/GameServer.cpp` L398~424
- **문제**: 시작 무기를 `pistol_9mm`로 지정하면서 `itemID = 1`로 설정했는데, `data/items.json`에서 `pistol_9mm`의 `"id"`는 `4`입니다. itemID로 아이템을 조회하는 코드가 있다면 잘못된 아이템을 반환합니다.
- **수정**: `startWeapon.itemID = 4;` (items.json 기준)

#### BUG-5. 좀비 드랍 아이템 key 오타 + 잘못된 itemID
- **파일**: `src/server/GameServer.cpp` L587~592
- **문제**: 좀비 사망 드랍에서 `item.key = "9mm_ammo"` 로 설정하는데 items.json의 실제 key는 `"ammo_9mm"`입니다. 또한 `item.itemID = 1000 + (rand()%1000)`은 items.json에 존재하지 않는 ID입니다.
- **수정**: key를 `"ammo_9mm"`으로, itemID를 `10`(items.json의 ammo_9mm id)으로 변경하세요.

#### BUG-6. 루트 박스 스폰에서 존재하지 않는 아이템 key 사용
- **파일**: `src/server/GameServer.cpp` L930~934
- **문제**: `spawnLootBoxes()`에서 `"9mm_ammo"`, `"rifle_556"`, `"556_ammo"` key를 사용하는데, 이 key들은 `data/items.json`에 **없습니다**. items.json의 실제 key는 `"ammo_9mm"`, `"pistol_9mm"` 등입니다.
- **수정**: items.json의 key 목록과 일치하도록 수정하세요.

#### BUG-7. `BuildSystem`의 재료 확인이 items.json과 불일치
- **파일**: `src/server/systems/BuildSystem.cpp` L42~44
- **문제**: 재료 key로 `"barricade_item"`, `"turret_item"`, `"wood_plank"`를 사용하는데, items.json에 이런 key는 없습니다. 실제로는 `"scrap_metal"`, `"plank"`, `"electronic_part"` 등을 조합하는 제작 레시피입니다. 결과적으로 재료를 못 찾아(`found = false`) **건설이 항상 실패**합니다.
- **수정**: 재료 확인 로직을 items.json의 `crafting_recipes`와 연동하거나, 아이템 key를 실제 존재하는 것으로 변경하세요.

#### BUG-8. `loadMap`에서 탈출존이 map.json을 무시하고 랜덤 배치
- **파일**: `src/server/GameServer.cpp` L699~706
- **문제**: map.json에 탈출존 좌표가 명시되어 있지만(`extractionZones`), 서버는 이를 읽지 않고 `rand() % 20 + 30` 타일에 임의 배치합니다. 클라이언트의 `m_extractionZones`(하드코딩된 4코너 좌표)와 서버의 실제 탈출존 위치가 **일치하지 않습니다**.
- **수정**: `m_map.getExtractionZones()`로 읽어서 `m_extraction.addZone()`에 전달하세요.

#### BUG-9. 플레이어 스폰 좌표가 맵 4분의 1 영역에만 집중
- **파일**: `src/server/GameServer.cpp` L374~377
- **문제**: 200×200 타일 맵인데 스폰 위치가 타일 `(2,2)`, `(116,2)`, `(2,116)`, `(116,116)`으로, 맵의 우하단 1/4은 아무도 스폰하지 않습니다. 4팀이 균형 있게 4코너에 배치되어야 합니다.
- **수정**: `spawnX = { 5*32, 190*32, 5*32, 190*32 }`, `spawnY = { 5*32, 5*32, 190*32, 190*32 }` 정도로 수정하세요.

#### BUG-10. `onAllianceChanged` 연합 해제 패킷 타입이 동일
- **파일**: `src/server/GameServer.cpp` L681~683
- **문제**: `active=true`와 `active=false` 두 경우 모두 `PacketType::S2C_AllianceAck`를 사용합니다. 클라이언트가 연합 성립과 해제를 구분할 수 없습니다.
- **수정**: `active=false`일 때 `S2C_AllianceAck`의 `active` 필드(L684)를 0으로 설정하는 것으로 충분하지만, 코드에서 삼항 연산자가 무의미하게 작성된 점만 정리하면 됩니다.

#### [개선-9] ZombieAI 시야 판정에 Line-of-Sight 추가
현재 `findNearestEnemy()`는 거리만 체크합니다. 벽 뒤에 있는 플레이어도 감지하는 문제가 있습니다. TileMap의 SOLID 타일을 따라 레이캐스팅을 추가하면 더 자연스럽습니다.

#### [개선-10] 좀비 순회 최적화 — 공간 해시 도입
`doAttack()`과 `findNearestEnemy()`가 매 틱 모든 엔티티를 O(N²)로 순회합니다. 좀비 15마리 × 플레이어 8명은 문제없지만, 좀비가 50+ 마리가 되면 병목이 생깁니다.

```
간단 개선: 검색 반경을 좁혀 early-exit (이미 600px 제한 있음, 유지)
본격 개선: 타일 기반 공간 해시로 근접 엔티티만 검색
```
